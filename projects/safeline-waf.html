Web Application Firewall Home Lab — SafeLine

SafeLine WAF · DVWA · Ubuntu · Reverse Proxy · Attack Simulation

Overview

Built a web application security lab to understand how a Web Application Firewall (WAF) protects applications from common web-based attacks. The lab focused on deploying SafeLine in front of a deliberately vulnerable application and validating detection and blocking behaviour.

Lab Setup

  • Deployed an Ubuntu-based web server hosting DVWA (Damn Vulnerable Web Application).
  • Configured SafeLine as a reverse proxy in front of the web application.
  • Ensured all client traffic passed through the WAF before reaching the application.

What I Did

  • Configured SafeLine security policies, including IP rules and rate limiting.
  • Generated attack traffic from a Kali Linux machine targeting known DVWA vulnerabilities.
  • Simulated common web attacks such as SQL injection and malicious request patterns.
  • Observed WAF decisions to understand how requests were allowed, challenged, or blocked.

What I Tested and Validated

  • Attack detection: verified SafeLine’s ability to identify malicious payloads.
  • Blocking effectiveness: confirmed that malicious requests were stopped before reaching the application.
  • Logging & visibility: reviewed WAF logs to correlate blocked requests with attack attempts.

Key Outcomes

  • Developed a practical understanding of how WAFs sit in front of applications and enforce security controls.
  • Gained experience interpreting WAF logs and understanding false positives vs real threats.
  • Strengthened knowledge of web attack patterns and defensive controls used in production environments.

Tools

  • SafeLine Web Application Firewall
  • DVWA (Damn Vulnerable Web Application)
  • Ubuntu Server
  • Kali Linux (attack simulation)
