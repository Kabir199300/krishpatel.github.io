SIEM, Endpoint & Monitoring Home Lab — Wazuh

Wazuh · Ubuntu Server · Windows Agents · File Integrity Monitoring · Authentication Monitoring

Overview

Built a home lab to practise SIEM-style monitoring and investigation. The goal was to centralise endpoint telemetry, generate realistic security events, and get comfortable triaging alerts and validating what “normal” looks like.

Lab Setup

  • Deployed Wazuh in a lab environment (server + dashboard components).
  • Onboarded endpoints using agents (Windows workstation as an endpoint source).
  • Configured core monitoring areas: file integrity monitoring (FIM) and authentication-related events.

What I Did

  • Created a baseline for normal endpoint behaviour (expected processes, logon patterns, file change activity).
  • Generated test activity to trigger alerts (unauthorised file changes, suspicious logon attempts, abnormal patterns).
  • Reviewed alerts and event details to understand why they fired and what evidence supports them.
  • Practised triage: severity judgement, false-positive checks, and what follow-up data would be needed.

What I Tested and Validated

  • FIM: confirmed alerts on monitored file changes and verified the “who/what/when” detail in events.
  • Authentication: validated visibility of login activity and identified suspicious patterns vs normal logons.
  • Signal vs noise: compared benign admin activity against suspicious changes to reduce false positives.

Key Outcomes

  • Improved confidence reading SIEM alerts and connecting them to real endpoint evidence.
  • Built a repeatable workflow for: baseline → generate event → validate alert → interpret severity.
  • Strengthened practical understanding of monitoring fundamentals used in SOC environments.

Tools

  • Wazuh
  • Ubuntu Server (lab host)
  • Windows endpoint (agent)
